Using The Command Line

Introduction

In this lab I explored the use of command line interfaces and terminal commands for file management and system navigation.

Command line tools are important because they allow users to interact directly with the operating system through text commands. During the lab I used PowerShell commands to navigate directories, create files and manage repositories.

Opening The Terminal

PowerShell was used as the command line environment during this lab. The terminal allows users to execute commands directly instead of relying only on graphical interfaces.

Through this exercise I became more confident using terminal environments and understanding how command line tools operate.

Directory Navigation

Different commands were used to move between folders and inspect directory contents.

Commands such as pwd, ls and cd were used regularly throughout the exercises. 

pwd
ls
cd TestFolder

These commands made it possible to move through the file system efficiently and understand the current working directory.

Creating Files And Folders

Commands were also used to create new folders and files directly from the terminal.

This demonstrated how command line tools can simplify file management tasks and automate workflows. 

mkdir TestFolder
touch test.txt

Learning these commands improved my understanding of how developers interact with systems using terminal environments.

Using Git Commands

Git commands were used to manage repositories and track changes within the project files.

Commands such as git status, git add and git commit were used to manage updates and upload changes to GitHub. 

git status
git add .
git commit -m "update"

Using Git through the command line improved my understanding of version control systems and repository management.

Reflection

This lab improved my confidence using terminal environments and command line tools. I learned how commands can be used to navigate systems, manage files and interact with repositories efficiently.

I also developed a better understanding of how developers use terminal environments in professional workflows and software development projects.
