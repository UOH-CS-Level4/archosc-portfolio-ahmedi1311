Accessing And Using A Virtual Machine

Introduction

In this lab I explored how virtual machines can be used for remote access and website hosting. The main aim of the lab was to understand how a remote server environment works and how websites can be deployed inside a virtual machine.

During the lab I learned about Remote Desktop Connection, PowerShell commands, Git installation and IIS web hosting services. I also explored how repositories can be cloned into a virtual environment and hosted through a web server.

Connecting To The Virtual Machine

The first part of the lab involved connecting to a university virtual machine using Remote Desktop Connection. This allowed access to a Windows Server environment through a remote connection.

The process involved entering the virtual machine hostname and authenticating using university login credentials. Once connected, the virtual machine desktop could be accessed and controlled remotely.

Through this process I gained a better understanding of remote access systems and how virtual machines are commonly used in development and hosting environments.

Using PowerShell And Git

PowerShell was used to run commands and manage files inside the virtual machine environment. Different commands were used to check software installations and manage repositories.

Git was used to clone and manage the portfolio repository inside the virtual machine. This allowed the website files to be copied directly from GitHub into the server environment. 

git version
git clone repository-link

Using Git inside the virtual machine improved my understanding of remote development workflows and repository management.

Hosting The Website Using IIS

IIS (Internet Information Services) was used to host the portfolio website inside the virtual machine. IIS is a web server platform provided by Microsoft for hosting websites and applications.

The website files were copied into the wwwroot directory so that IIS could detect and host the portfolio automatically.

PowerShell commands were also used to check whether the IIS service was active and functioning correctly. 

Get-Service w3svc

This section of the lab helped me understand how websites can be hosted on remote servers and how web server software operates in a virtual machine environment.

Reflection

This lab improved my understanding of virtual machines, remote access and server hosting environments. I learned how repositories can be deployed remotely and how server software such as IIS can be used to host websites.

I also developed a better understanding of remote workflows, PowerShell usage and the importance of virtualisation technologies in modern computing systems.
