OS Resource Allocation

Introduction

In this lab I explored operating system resource allocation and deadlock management techniques.

Operating systems must manage resources such as memory, files and CPU access between multiple processes safely .

Deadlocks

A deadlock occurs when multiple processes wait indefinitely for resources that are being held by each other.

During the exercises I identified deadlock situations and explored the four conditions required for a deadlock to occur:

  • Mutual Exclusion
  • Hold and Wait
  • No Preemption
  • Circular Wait

Safe And Unsafe States

A safe state occurs when the operating system can allocate resources in a way that allows all processes to complete successfully.

Unsafe states increase the risk of deadlocks because resource allocation may prevent processes from finishing execution.

During the lab I analysed allocation tables and determined whether systems were in safe or unsafe states.

Banker's Algorithm

The Banker's Algorithm is used to determine whether a system is in a safe state before allocating resources to processes.

During this exercise I completed resource allocation tables and calculated available resources, maximum resource requests and safe execution sequences.

The algorithm helps prevent deadlocks by refusing unsafe resource allocation requests.

Resource Allocation Tables

Resource allocation tables were used to organise process information and resource usage during the exercises.

These tables helped visualise:

  • Allocated Resources
  • Available Resources
  • Maximum Resource Requests
  • Remaining Resource Needs

Resource allocation tables are useful because they simplify deadlock analysis and safe state calculations.

Reflection

I also improved my ability to analyse resource allocation tables and determine whether systems are operating safely.
